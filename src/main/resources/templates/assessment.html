Plain-Text Input Guide

How plain-text assessment works

Plain-text input allows the user to describe the organization’s defensive capabilities and threat context using simple natural language, without following a strict JSON structure or a specific syntax. This makes the assessment easier to use for both security specialists and users who are not familiar with the internal defensive capability fields of the platform.

D3FENDer processes the submitted text by identifying phrases and terms that correspond to entries in its defensive capabilities registry, security controls registry, and threat context model. The extracted information is then normalized into the same internal structure used by the JSON assessment flow.

Example input

We are a software engineering company. We have token based authentication.
We do not have token binding. We also perform code scanning.

1. Text processing

The submitted paragraph is split into smaller informational units. D3FENDer detects positive and negative statements, such as controls that exist and controls that are explicitly missing.

2. Normalization

The detected phrases are mapped to canonical internal fields. For example, “code scanning” can be normalized as sw.code_analysis.

3. Rule evaluation

The normalized capabilities are evaluated against assessment rules. When a relevant protection is missing, D3FENDer generates a security finding.

Semantic matching

Plain-text input does not require the exact internal field names. D3FENDer uses a lightweight semantic layer to compare user phrases with known capability descriptions and aliases. This means that similar expressions such as “code scanning”, “code analysis”, or “code review tooling” can be matched to the closest normalized defensive capability.

Useful phrases to include

Organization context

  • We are a software engineering company.
  • We use web applications.
  • We expose services to the internet.
  • We use cloud infrastructure.

Security capabilities

  • We use token based authentication.
  • We perform code scanning.
  • We do not have token binding.
  • We have multi-factor authentication.
